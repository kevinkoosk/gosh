GO.sh: Simple Accounting using HAML and CSV

Description

  1. Simple accounting for my personal use. Generates HTML from HAML and CSV.
  2. Individual records are stored in CSV and HAML files. They must be in same directory. CSV file contains transactions. HAML file contains client's record (long content possible).
  3. Client's name is in first line of HAML file. First line should be something like ~CLIENT'S NAME. The client's name comes after the tilde (~).

Usage

./GO.sh directory_containing_data
  1. If records are in example directory, the command to process them is: 
    ./GO.sh example
  2. GO.sh will generate table.html and new files in out. To distribute the generated files, copy table.html, out directory and script directory.
  3. Keep different sets of data in their own directories, e.g.
    • office_accounts = for office accounts
    • house_expenses = for household expenses.
  4. To view records in web browser, enter: 
    ./GO.sh example
    firefox table.html
  5. You can view the generated files anywhere using: (a) table.html and (b) two directories of out and script.

Notes

  1. Contents of out will be deleted everytime GO.sh is run.
  2. GO.sh uses Linux, Python and the HamlPy package.
  3. Banks do end of day processing, so take a hint. You only need to regenerate HTML as needed.
  4. Records are managed by editing CSV and HAML.
